Reports classes that can be marked as light services using the @Service annotation instead of being registered as services in plugin.xml

A service not going to be overridden does not need to be registered in plugin.xml. Instead, annotate service class with @Service. Project-level services must specify @Service(Service.Level.PROJECT).

Restrictions:

  • The version of the IntelliJ Platform is not less than 2019.3
  • Service class must be final.
  • serviceInterface is not specified.
  • If service is a PersistentStateComponent, roaming must be disabled (roamingType = RoamingType.DISABLED).
  • preload is not specified.

See Services in IntelliJ Platform Plugin SDK docs for more details.

See also the Plugin DevKit | Plugin descriptor | A service can be converted to a light one inspection.
